# Design

智学闯关的目标视觉系统。register = **product**：界面服务于任务，游戏感集中在「奖励瞬间」，不铺满全屏。
本文件是「整体视觉升级」的目标态，刻意区别于当前皮肤（candy 紫 `#8b5cf6`+amber、`radial-gradient` body 背景、满屏 `white/80` 卡片、24px+ 圆角）。实现时以本文件为准。

## 设计方向

精致克制的闯关感。三词：进取、清醒、有温度。
- 颜色策略：**Restrained → Committed**。中性表面打底，**一个主色（靛蓝）**承载产品与操作，**金色严格只用于奖励**，绿/红为标准语义态。
- 反面（绝不出现）：紫色渐变、glassmorphism、满屏等大卡片、每段顶着小写字母间距 eyebrow、gradient text、ghost-card（1px 边框 + 大投影并用）、24/32px+ 大圆角、儿童化气球感。

## Color

全部 OKLCH。角色绑定——颜色由语义决定，不做装饰。

### 中性表面（冷调近白，微微偏主色，**不偏暖、不奶油**）
- `--bg`          oklch(0.985 0.004 265)  应用底色，**纯色，无 radial 渐变**
- `--surface`     oklch(1 0 0)            卡片/面板主表面（白）
- `--surface-2`   oklch(0.972 0.006 265)  次级层：侧栏、工具条、嵌套区
- `--border`      oklch(0.918 0.008 265)  1px 发丝边框
- `--ink`         oklch(0.255 0.022 265)  正文/标题（白底对比 ≥ 12:1）
- `--muted`       oklch(0.475 0.020 265)  次要文字（白底 ≥ 4.6:1，过 AA；**替换当前偏弱的 #6b6783**）

### 主色 · 靛蓝（产品身份、主操作、当前态、进度填充、导航选中）
比当前 candy 紫更深、更冷、更克制。
- `--brand-50`   oklch(0.970 0.018 265)
- `--brand-100`  oklch(0.930 0.040 265)
- `--brand-200`  oklch(0.870 0.070 265)
- `--brand-500`  oklch(0.560 0.165 265)
- `--brand-600`  oklch(0.500 0.180 265)   主按钮底色（白字对比 ≥ 4.5:1，必须在浏览器校验）
- `--brand-700`  oklch(0.430 0.165 265)   hover/按下

### 奖励 · 金（**仅**积分、通过徽章、top3、勋章、得分揭晓的庆祝；绝不做普通装饰）
- `--gold-100`   oklch(0.945 0.045 85)
- `--gold-500`   oklch(0.800 0.150 85)
- `--gold-600`   oklch(0.720 0.150 78)    金底配深 ink 文字（金底不要配白字，对比不足）

### 语义态
- `--success`    oklch(0.640 0.150 160)   通过/✓/成功 toast（绿）
- `--danger`     oklch(0.580 0.200 25)    错误/驳回/删除（红）
- `--warning`    `--gold-600` 复用

> 色盲安全：通过/失败不能只靠绿/红区分，必须同时带图标或文案（✓ / 数字 / 「未达 60」）。

### 状态词汇（每个交互组件都要齐全）
default / hover / focus-visible / active / disabled / selected / loading / error。
- focus-visible：`2px` `--brand-600` 实色 ring + `2px` 偏移，键盘可见。
- disabled：降透明度 + 去饱和，不用纯灰糊死。

## Typography

product register：**一套 sans 通吃** UI（标题/正文/按钮/标签/数据）。
- UI：`Geist`（已装）→ 回退 `PingFang SC`/`Microsoft YaHei`/system-ui。Geist 的等宽数字很适合分数/积分。
- 等宽：`Geist Mono`（**新增**）仅用于成长画像的 git 风格按行 diff 与代码态。
- 共两族，符合上限。

固定 rem 梯度（**不用 fluid clamp**，product UI 在稳定 DPI 下查看）：
- display（仅得分揭晓的大数字）48–56px / 800
- h1 30px / 700 · h2 22px / 700 · h3 18px / 600
- body 16px / 400 · body-sm 14px · caption 12px
- 梯级比 ~1.2；字重拉层次而非堆字号。
- 中文不挤：标题 letter-spacing 0；正文行高 1.7；散文行宽 ≤ 75ch（表格/密集 UI 可更密）。
- 不用全大写正文；不用 gradient text。

## Spacing & Layout

- 页面框架（统一容器，PC 端大气、随屏自适应、与页头对齐）：
  - `.page` = max-w 1152px（72rem）+ 响应式内边距（16/24/32px）。用于栅格/列表/仪表盘/页头。
  - `.page-narrow` = max-w 896px（56rem）。用于阅读/表单单列（写笔记、反思、资料、成长、兑换），控制行宽可读。
  - 登录/改密保留聚焦居中卡片（max-w-md），不入框架。
  - 列表密集页在桌面用 2 列布局（排名/审批/排行榜/统计），让宽度成为有意的留白而非拉伸。
- 基准 4px 栅格：4 / 8 / 12 / 16 / 24 / 32 / 48 / 64。
- 圆角：卡片/面板/输入 **12–14px**（不要 24/32+）；tag/小徽章可 full-pill；按钮 10–12px。
- 响应是结构性的：闯关地图在窄屏单列、PC 多列；移动端输入区与主按钮可达、够大（命中区 ≥ 44px）。
- 卡片是「在合适时才用」的选择，不是默认容器；**绝不嵌套卡片**。
- 海拔二选一，不叠加：要么 1px `--border`，要么一层 ≤ 8px 模糊的柔和阴影；**不要边框+大投影并用**（ghost-card）。
- z-index 语义梯度：dropdown < sticky < modal-backdrop < modal < toast < tooltip。

## Components（统一词汇，跨页一致）

- **按钮**：primary（`--brand-600` 实底白字）/ secondary（`--surface` + 1px border + ink）/ ghost（透明 + ink）/ danger。形状、圆角、内距全站一致。标签是「动词+宾语」（提交笔记 / 通过审批 / 兑换积分）。
- **输入/文本域**：写笔记是核心任务——大号、舒适行高、清晰字数计数（100–5000，越界禁用提交并提示），focus 态明显。
- **进度**：章节进度条用 `--brand`（非渐变堆叠）；闯关地图节点用「数字（已解锁）/ 锁（未解锁）/ ✓（已通过）」清晰三态。
- **奖励元件**（金 + 动效，集中投放）：积分数字、通过徽章、top3、勋章、得分揭晓 count-up + 庆祝。
- **空态**：教用户怎么用，不是「暂无数据」。
- **加载**：内容区用骨架屏，不用居中转圈。
- 不发明非标准控件（自定义滚动条、怪异表单、非标准 modal）；modal 是最后手段，优先就地/渐进展开。

## Motion

- 常规 UI 过渡 **150–250ms**，ease-out（quart/quint），无 bounce/elastic。
- **奖励瞬间是刻意的例外**：得分 count-up、pop、积分到账、升章解锁 700–900ms，是产品情绪核心，但只在这些节点出现，不滥用。
- 列表可错峰入场（stagger）；不要给每个区块套同一套入场反射。
- 揭示动画必须增强「已可见」的默认态，不能把内容可见性 gate 在 class 触发的 transition 上（隐藏标签页/无头渲染会导致空屏）。
- **减弱动效**：每个动效都要 `@media (prefers-reduced-motion: reduce)` 降级（淡入或直接显示），confetti/count-up 在该模式下直接显示终值。

## Accessibility

- 正文/控件对比 ≥ 4.5:1（大字/粗体 ≥ 3:1），placeholder 同样达标；不靠浅灰「显高级」——实现时逐页在浏览器校验。
- 键盘可达：提交笔记、答追问、兑换、审批、管理操作可纯键盘完成，focus-visible 清晰。
- 通过/失败不只靠颜色，配图标+文案。
- 全站中文；语义 HTML + 必要 ARIA。
