# 演示与验证指南

## 一键起步

```bash
pnpm install
pnpm db:up          # 起本地 Postgres（Docker，主机 5433）
pnpm db:migrate     # 建表
pnpm db:seed        # 灌演示数据（账号 + AI 100 词 + 进度/积分/兑换/排名演示）
pnpm dev            # http://localhost:3000
```

- 打分默认 `mock`（秒出分、不花钱）。要用真 DeepSeek：`.env` 设 `SCORING_PROVIDER=deepseek` + 填 `DEEPSEEK_API_KEY`，重启 `pnpm dev`。
- 演示阶段可关掉强制改密：`.env` 设 `REQUIRE_PASSWORD_CHANGE=false`。

## 完整演示路径（端到端）

1. **学习闭环（员工）**：张三登录 → 填资料 → 闯关地图（第 1–3 关开放、4–5 锁定）→ 进第 1 关 → 任一关键词 → 写 ≥100 字笔记 → AI 初评 + 追问 → 作答 → 终评（≥60 记 1 积分；<60「再来一次」取最高分）→ 完成后页面出现「本词全员均分 / 你超过 %」+「看看别人怎么写的」同伴笔记。
2. **成长轨迹**：李四 → 顶栏「成长」→ 线性卡片时间线，每张卡显示触发关键词、新增标签、画像 **git 行 diff**（画像随学习越来越清晰）。
3. **章节反思**：赵六 → 第 1 关 → 顶部「来做章节反思」→ 3 个结合其岗位的反思题 → 作答 → AI 总结 + 画像新增「章节反思」小节。
4. **排行榜**：员工顶栏「排行榜」→ 积分榜 + 各章 top3 冠军。
5. **积分兑换**：李四 → 顶栏「兑换」→ 看余额、发起兑换；超管 → 「兑换审批」→ 通过李四的待审申请 → 积分扣减。
6. **排名结算（管理员）**：超管 → 「排名」→ 第 1 章「结算本章排名」→ 赵六/钱七/孙八/周九 出榜，top3（含并列）各 +100。
7. **周期演示**：超管 → 后台「快进一周 / 回退一周」→ 闯关地图解锁进度随之变化。
8. **运营管理**：超管 → 「名单」（加人/导入/重置密码/改角色）、「内容」（切换学科/开始日/编辑关键词）、「数据统计」（进度/质量/财务）。

## 自动化测试

- `pnpm test` —— 单元测试（22 个，确定性，无需 DB）：打分 rubric、Mock 确定性、个性化记忆、画像 git-diff、周解锁、章节反思。
- `pnpm test:int` —— 集成测试（真本地 DB + Mock 评分器，自建测试用户并自清理）：学习闭环「提交→打分→追问→终评→记分」、重提取最高分不重复发分、画像快照。
- `pnpm typecheck` / `pnpm build` —— 类型检查与生产构建。

> 兑换并发安全、排名结算冻结快照等「碰钱」逻辑，开发期以 Next 运行时内的临时路由做过并发/幂等端到端验证（详见提交记录与对抗式审查结论）。
